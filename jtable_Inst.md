# Shift Table & Peak Annotator for TopSpin 5.0+ (Jython)

## Обзор проекта
Скрипт добавляет в интерфейс TopSpin встраиваемую панель **«Shift Table & Peak Annotator»**, которая позволяет:
- Вести таблицу химических сдвигов (остатки → протоны/углероды) в привязке к папке образца.
- Автоматически загружать/сохранять сдвиги в XML-файл `shift_data.xml` внутри выбранной директории образца.
- Выбирать 1D‑эксперимент из образца и аннотировать пики в его `peaklist.xml`, используя данные из таблицы.

Панель встраивается **справа от рабочей области спектра** как дополнительный контейнер в `BorderLayout.EAST` главного окна TopSpin.

## Интерфейс панели
- **Верхняя строка:** Sample (имя папки) + кнопка Browse для выбора папки образца + кнопка Close Panel.
- **Таблица:** столбцы `Residue`, `H1/C1`…`H6/C6`. Чётные строки – протоны, нечётные – углероды (имя остатка автоматически копируется из протонной строки).
- **Нижняя панель кнопок (с переносом):**
- `Load Shifts` – перезагрузить `shift_data.xml` из выбранной папки.
- `Save Shifts` – сохранить изменения.
- `Annotate Peaks` – выбрать эксперимент и открыть Pick-окно для аннотирования.
- `Add Residue` – добавить пустую пару строк.
- `Copy All` – скопировать таблицу в буфер обмена.
- `Clear All` – очистить таблицу.
- `Remove Selected` – удалить выделенные пары.
- `Move Up` / `Move Down` – переместить выделенную пару.
- `Export CSV` – сохранить таблицу в CSV в папке образца.

## Рабочий процесс аннотирования
1. Нажмите **Browse**, выберите папку образца (где лежат подпапки `1`, `2`, … с `acqu`).
2. Заполните сдвиги в таблице (или загрузите из существующего `shift_data.xml`).
3. Нажмите **Annotate Peaks** → откроется диалог со списком экспериментов.
4. Выберите 1D‑эксперимент (например, `Exp 1 (1H)`). Если выбран 2D, появится предупреждение.
5. Откроется **Pick-окно** с отфильтрованными строками (протоны для ¹H, углероды для ¹³C). Выделите нужные ячейки.
6. Нажмите **Apply Annotations**. Сейчас здесь ошибка: аннотации не формируются (см. раздел «Текущие проблемы»).

## Архитектура скрипта (ключевые классы)
- `WrapLayout` – менеджер размещения, переносящий кнопки на новую строку.
- `MergedTableModel` – модель таблицы с валидацией и автоматическим копированием имени остатка.
- `ResidueRenderer` / `PairSplitRenderer` – кастомные рендереры для раскраски пар строк.
- `ShiftXMLHandler` – чтение/запись XML.
- `ShiftTablePanel` – главная панель, содержит всю логику.
- `find_parent_container` / `embed_panel` – функции для поиска контейнера в интерфейсе TopSpin и встраивания панели.

## Важные особенности Jython в TopSpin
- **Конфликты имён:** `getName()` и другие методы с одинаковыми именами в Java/Python могут требовать явного вызова Java-метода. Используйте `component.getClass().getName()` вместо `component.getClass().getName()`.
- **Отсутствие некоторых модулей:** `StringSelection` нужно импортировать из `java.awt.datatransfer`, `File` из `java.io`.
- **Не все стандартные методы Swing работают:** `JTable.getSelectedCells()` недоступен, используем `getSelectedRows()` + `getSelectedColumns()`.
- **`TableRowSorter` может быть нестабилен** – вместо фильтрации строк через сортировщик строим отдельную модель данных.
- **Импорты:** избегайте `from java.awt import Dimension`, используйте `from java.awt import Dimension as _Dimension` или прямой `java.awt.Dimension`, чтобы избежать конфликтов.

## Диагностика и отладка
При нажатии **Apply Annotations** в консоль выводятся отладочные сообщения:
- `Pick model rows` – число строк в модели Pick-окна.
- `first row of pick model` – данные первой строки.
- `selected rows (view)` / `selected cols (view)` – индексы выделенных ячеек.
- `annotations` – список сформированных пар (сдвиг, метка).

Если `annotations` пуст – значит, на каком‑то этапе отсеиваются все значения. Возможные причины:
- `residue` пустой (не заполнено имя остатка в таблице).
- `shift_str` пустой или не числовой.
- Неверное преобразование индексов столбцов.

## Что нужно исправить (для агента)
1. Убедиться, что в Pick-окне корректно отображаются значения сдвигов (диагностика первой строки это покажет).
2. Проверить, что `pick_model.getValueAt(pick_row, col)` возвращает ожидаемое строковое представление числа.
3. Возможно, проблема в том, что `pick_table.convertColumnIndexToModel(pick_col)` возвращает не тот индекс – сравнить с исходными номерами столбцов.
4. Если сдвиги на месте, но аннотации всё равно не создаются – проверить, не перепутан ли `is_proton` для выбранного эксперимента (логи `selected rows` помогут).
